Introduction

NeonScript is a lightweight, high-performance wrapper designed for modern web applications. It simplifies DOM manipulation while maintaining a tiny footprint.

The core philosophy is based on three pillars:

  • Speed
  • Clarity
  • Extensibility
Getting Started

To begin using NeonScript, you can include the CDN link in your HTML header. Ensure you are using the latest stable version.

npm install neonscript-core

Once installed, initialize the library in your entry point file to gain access to the global object.

Variable Declaration

NeonScript uses a strict scoping system. You should define your variables using the specific keywords provided by the engine.

neon let user = "Admin";

This ensures that variables are tracked efficiently by the garbage collector.

  • Immutable constants
  • Reactive signals
Function Syntax

Functions in NeonScript are first-class citizens. They can be passed as arguments and returned from other functions.

fn calculate(x, y) { return x + y; }

You can also use arrow-shorthand for anonymous callbacks within your logic.

Async Operations

Handling data fetching is streamlined through the internal promise engine. Use the keyword awaiting to pause execution.

val data = awaiting fetchSource(url);

Error handling is managed through a global observer pattern, reducing the need for repetitive try-catch blocks.

  • Automatic retries
  • Timeout management
